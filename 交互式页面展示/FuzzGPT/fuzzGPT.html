论文标题: Large Language Models are Edge-Case Generators: Crafting Unusual Programs for Fuzzing Deep Learning Libraries

简称: FuzzGPT

核心思想：从“有效”到“不寻常”的进化

FuzzGPT 是对 TitanFuzz 思想的重大升级。TitanFuzz 证明了LLM能生成有效的测试代码，但这些代码往往是“普通”的。FuzzGPT 的核心目标是引导LLM生成不寻常的 (unusual)、能触发边缘情况的测试代码，因为这才是Fuzzing发现深层漏洞的关键。

学习要点

1. 核心问题：普通测试用例的局限性

直接使用预训练LLM生成的代码（如TitanFuzz）往往会模仿其训练数据中的常见模式。然而，许多顽固的Bug隐藏在不常见的API用法或边缘的输入组合中。Fuzzing的威力恰恰体现在探索这些“不寻常”的路径上。

2. 核心思路：利用历史Bug“投喂”LLM

FuzzGPT的解决方案非常巧妙：让LLM从过去已知的Bug中学习，从而学会如何创造新的Bug。 它基于一个经典假设：历史上触发Bug的代码包含了发现新Bug的宝贵“基因”（例如，特殊的数据类型、不寻常的参数组合等）。

3. 方法论：FuzzGPT框架

工作原理：一个最小化的示例

一个核心问题是：FuzzGPT生成的“坏”代码是如何用来测试的？关键在于理解：FuzzGPT生成的代码是测试用例 (Test Case)，其测试目标是它所调用的底层库 (Library)。

让我们通过一个假设的 SafeMath 库来理解这个过程。

第一步：被测试的目标库

假设我们有一个 SafeMath 库，其中包含一个计算倒数的函数。开发者可能没有考虑到所有边缘情况。

# SafeMath 库的内部代码 (被测试的对象)
def safe_inverse(x):
  if x == 0:
    raise ValueError("Cannot compute inverse of zero.")
  # 开发者可能忽略了极小浮点数会导致溢出的问题
  return 1.0 / x

第二步：FuzzGPT学习历史Bug的“基因”

FuzzGPT在其数据集中学到了一个模式：“使用非常接近于零的极小浮点数作为除数，可能触发边缘Bug”。

第三步：FuzzGPT生成“坏”代码（测试用例）

现在，我们让FuzzGPT为 safe_inverse 函数生成测试用例。它不会生成普通用例（如 safe_inverse(10)），而是会利用学到的“基因”生成一个不寻常的用例：

# FuzzGPT 生成的“坏”代码 (它本身是测试用例)
import SafeMath

# 使用了从历史Bug中学到的“极小浮点数”模式
test_value = 1e-308 # 一个非常接近于零的数

print(f"Testing with unusual value: {test_value}")

# 调用我们想要测试的库函数
result = SafeMath.safe_inverse(test_value)
print(result)

第四步：执行测试用例，发现库的Bug

当我们执行上面由FuzzGPT生成的代码时，SafeMath.safe_inverse(test_value) 会尝试计算 1.0 / 1e-308。这个结果会超出标准浮点数的表示范围，从而导致程序因浮点数溢出 (FloatingPointError) 而崩溃 (Crash)。

这个崩溃清晰地证明了我们的 SafeMath 库存在一个Bug：它的 safe_inverse 函数没有正确处理极小浮点数输入。通过这种方式，FuzzGPT成功地利用生成的“坏”代码，发现了底层库的隐藏缺陷。

总结：“坏”代码如何发现Bug

基于以上示例，我们可以将FuzzGPT的工作机理总结为以下四点：

  • 语法正确性：FuzzGPT生成的测试用例在语法上是完全正确的Python代码。
  • 语义的“坏”：其“坏”体现在语义层面——它以非典型、极端、且借鉴了历史Bug模式的方式调用库函数，专门探测API的薄弱环节。
  • 测试即执行：测试过程就是直接执行这段由FuzzGPT生成的代码。
  • Bug的显现：如果被测试的库在处理这种极端调用时发生崩溃、返回错误结果或挂起，这就标志着一个潜在的Bug被成功发现。

主要创新与启示

  1. 开创“边缘案例生成”新方向

    首次展示了如何主动引导LLM从生成“普通”代码转向生成“不寻常”的边缘案例代码，极大地提升了Fuzzing的效率和深度。

  2. 实现全自动化的“历史驱动Fuzzing”

    将一个依赖大量人工经验和规则的经典Fuzzing思想，通过LLM的能力实现了端到端的自动化，这是一个重大的工程和理念突破。

  3. 展示了“驾驭”LLM的巨大潜力

    证明了通过提供高质量的上下文或进行领域内的微调，可以有效“驾驭”LLM，使其生成更符合特定任务（如寻找Bug）要求的内容。

  4. 对AI安全领域的深刻启示

    FuzzGPT不仅是一个强大的防御性工具，它也揭示了LLM在安全领域的双重性。理解如何引导LLM生成“坏”代码（用于测试），对于未来防御由AI驱动的、能自动寻找并利用漏洞的攻击至关重要。
